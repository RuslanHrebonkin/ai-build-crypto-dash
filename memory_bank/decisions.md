# Decisions Log

## D-001: Contracts-First

- **Решение**: начать проект с фиксации контрактов API (`swagger.json`) и схемы БД (`db_schema.sql`).
- **Причина**: уменьшить интеграционные риски между frontend и backend.

## D-002: Трехслойная архитектура

- **Решение**: разделить проект на `frontend`, `backend`, `db`.
- **Причина**: простота масштабирования и независимых релизов.

## D-003: TypeScript как обязательный стандарт фронтенда

- **Решение**: использовать строгую типизацию API-ответов.
- **Причина**: раннее обнаружение ошибок на этапе разработки.
