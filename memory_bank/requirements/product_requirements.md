# Product Requirements

## Цель продукта

Веб-приложение предоставляет пользователю защищенный личный кабинет для ведения собственного watchlist криптовалютных торговых пар (например, BTC/USDT).

## Ключевые бизнес-требования

- Пользователь авторизуется через email/пароль.
- После авторизации пользователь получает доступ только к своим данным.
- У каждого пользователя есть индивидуальный список торговых символов.
- Пользователь может выполнять CRUD-операции над символами:
  - **Create**: добавить новый тикер в watchlist.
  - **Read**: просматривать полный список своих тикеров.
  - **Update**: редактировать существующий тикер.
  - **Delete**: удалять тикер из списка.

## Ограничения и ожидания

- Изоляция данных между пользователями обязательна.
- API и БД-контракты должны быть стабильными до начала реализации.
- Фронтенд должен иметь предсказуемую типизацию ответов API.
